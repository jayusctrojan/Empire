# Empire Specifications

This directory contains GitHub OpenSpec-compliant specifications for the Empire v7.2 project.

## Directory Structure

```
specs/
└── 1-empire-v7.2/          # Main Empire v7.2 specification
    ├── spec.md             # Feature specification (entry point)
    ├── plan.md             # Implementation plan (to be created)
    └── tasks.md            # Actionable tasks (to be created)
```

## How This Relates to Existing Documentation

The Empire project has extensive existing documentation that predates GitHub OpenSpec. Rather than duplicating content, the OpenSpec structure **references** the existing documentation:

### Existing Documentation (Unchanged)

- **`/srs/`** - Software Requirements Specification (11 detailed files, 340+ requirements)
- **`/workflows/`** - Milestone-based implementation guides (19 files, 8 milestones)
- **`/docs/`** - Additional documentation (architecture, guides, services, analysis)
- **`/empire-arch.txt`** - Core architecture specification (1,352 lines)
- **`/README.md`** - Project overview and quick start
- **`/claude.md`** - AI assistant and MCP tools guide

### OpenSpec Structure (New)

- **`/specs/1-empire-v7.2/spec.md`** - **Entry point** specification that references all existing docs
- **`/.claude/commands/`** - Slash commands for Claude (automatically generated by specify init)
- **`/.specify/`** - OpenSpec configuration and templates

## Workflow: How to Use OpenSpec with Empire

### 1. Overview Phase
Start with `/specs/1-empire-v7.2/spec.md` to understand:
- High-level architecture
- 8 milestones
- Key components
- References to detailed docs

### 2. Deep Dive
Follow references to detailed documentation:
- **Requirements**: `/srs/03_specific_requirements.md`
- **Architecture**: `/empire-arch.txt`
- **Milestones**: `/workflows/milestone_*.md`
- **Schemas**: `/workflows/database_setup.md`

### 3. Planning & Tasks
Use OpenSpec commands to create implementation artifacts:
- `/speckit.plan` - Generate implementation plan
- `/speckit.tasks` - Create actionable tasks
- `/speckit.implement` - Execute implementation

## OpenSpec Commands Available

With the GitHub OpenSpec integration, you now have these slash commands available in Claude:

1. **`/speckit.constitution`** - Establish project principles
2. **`/speckit.specify`** - Create/update specifications
3. **`/speckit.plan`** - Generate implementation plans
4. **`/speckit.tasks`** - Create actionable task lists
5. **`/speckit.implement`** - Execute implementation
6. **`/speckit.clarify`** (optional) - Ask clarifying questions
7. **`/speckit.analyze`** (optional) - Cross-artifact consistency check
8. **`/speckit.checklist`** (optional) - Generate quality checklists

## Adding New Features

When adding a new feature to Empire:

1. Run `/speckit.specify "feature description"`
   - This creates a new `specs/N-feature-name/` directory
   - Generates `spec.md` for the new feature

2. Run `/speckit.plan` to create an implementation plan

3. Run `/speckit.tasks` to generate actionable tasks

4. Implement the feature following the tasks

5. Update the relevant milestone documentation in `/workflows/`

## Benefits of This Hybrid Approach

1. **Preserves existing work** - All comprehensive SRS and workflow docs remain intact
2. **Adds modern tooling** - OpenSpec provides structured workflows for new features
3. **Single entry point** - `/specs/1-empire-v7.2/spec.md` gives quick overview
4. **AI-friendly** - Claude can navigate between OpenSpec and detailed docs seamlessly
5. **Incremental adoption** - Use OpenSpec for new features while keeping existing docs
